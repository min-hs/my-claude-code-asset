# Review 커맨드

코드 리뷰를 수행합니다.

## 실행 순서

1. 변경사항 확인
```bash
git diff
git diff --stat
```

2. 코드 리뷰 체크리스트

### 기능
- [ ] 요구사항 충족 여부
- [ ] 엣지 케이스 처리
- [ ] 에러 핸들링

### 코드 품질
- [ ] 함수 50줄 이하
- [ ] 파일 800줄 이하
- [ ] 매직넘버 없음 (#define/const 사용)
- [ ] 네이밍 규칙 준수

### 메모리 (임베디드 필수)
- [ ] 동적 메모리 할당 후 해제
- [ ] 버퍼 오버플로우 가능성
- [ ] 널 포인터 체크
- [ ] 메모리 정렬 이슈

### 안전성
- [ ] 인터럽트 안전성 (critical section)
- [ ] 재진입 가능성
- [ ] 타이밍 이슈

### 스타일
- [ ] 함수명: snake_case
- [ ] 구조체: PascalCase
- [ ] 매크로: UPPER_SNAKE_CASE
- [ ] 한국어 주석

3. 정적 분석
```bash
cppcheck --enable=all --inconclusive src/
```

## 리뷰 결과 형식
```markdown
## 코드 리뷰 결과

### ✅ 좋은 점
- 항목

### ⚠️ 개선 필요
- 파일:라인 - 이슈 설명

### ❌ 반드시 수정
- 파일:라인 - 이슈 설명
```

## 주의사항
- 비판보다 개선 제안 중심
- 심각도 표시 (필수/권장/선택)
