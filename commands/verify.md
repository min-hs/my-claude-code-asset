# Verify 커맨드

빌드, 정적분석, 코드 품질을 검증합니다.

## 실행 순서

1. 클린 빌드
```bash
make clean && make
```

2. 정적 분석
```bash
cppcheck --enable=all --inconclusive --error-exitcode=1 src/
```

3. 코드 포맷 확인
```bash
clang-format --dry-run --Werror src/*.c include/*.h
```

4. 메모리 체크 (가능한 경우)
```bash
valgrind --leak-check=full ./build/app
```

## 검증 체크리스트

### 빌드
- [ ] 컴파일 에러 없음
- [ ] 컴파일 워닝 없음
- [ ] 링크 에러 없음

### 정적 분석
- [ ] cppcheck 통과
- [ ] 널 포인터 역참조 없음
- [ ] 버퍼 오버플로우 없음
- [ ] 메모리 누수 없음

### 코드 품질
- [ ] 매직넘버 없음
- [ ] printf 디버깅 제거
- [ ] 주석 작성 완료
- [ ] 네이밍 규칙 준수

### 커밋 준비
- [ ] 불필요한 파일 제외 (.gitignore)
- [ ] 바이너리 파일 제외
- [ ] 민감 정보 없음

## 검증 결과 형식
```markdown
## 검증 결과

### 빌드
- 상태: ✅ 통과 / ❌ 실패
- 워닝: 0개

### 정적 분석
- 상태: ✅ 통과 / ❌ 실패
- 이슈: 0개

### 코드 품질
- 상태: ✅ 양호 / ⚠️ 개선필요

### 총평
(커밋 가능 여부 및 권장사항)
```

## 주의사항
- PR 전에 반드시 실행
- 워닝도 가능하면 제거
- 검증 실패 시 커밋 금지
