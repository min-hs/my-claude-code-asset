# Plan 커맨드

작업 계획을 수립합니다.

## 실행 순서

1. 요구사항 분석
- 사용자 요청 파악
- 관련 파일/코드 확인
- 제약사항 확인

2. 계획 수립
```markdown
## 작업 계획

### 목표
(달성하려는 목표)

### 분석
- 현재 상태
- 필요한 변경사항
- 영향 범위

### 단계별 계획
1. [ ] 단계1 - 예상 시간
2. [ ] 단계2 - 예상 시간
3. [ ] 단계3 - 예상 시간

### 관련 파일
- `파일1` - 역할
- `파일2` - 역할

### 리스크
- 리스크1 및 대응방안
- 리스크2 및 대응방안

### 완료 기준
- [ ] 기준1
- [ ] 기준2
```

3. 사용자 확인 요청

## 임베디드 개발 체크포인트
- 메모리 제약 확인
- 인터럽트 영향 검토
- 타이밍 요구사항 확인
- 하드웨어 의존성 파악
